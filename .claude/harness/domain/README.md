# domain/ — 프로젝트 특화 지식 베이스

> 이 폴더는 새 프로젝트 시작 시 채워야 하는 유일한 폴더입니다.
> agents/, rules/, templates/, skills/ 는 그대로 재사용됩니다.

---

## 필수 파일

### `strategy.md`
프로젝트 전략 문서. 다음 항목을 포함해야 합니다:
- 서비스 개요 (한 문단)
- 타겟 유저 정의
- 핵심 KPI
- Phase별 목표
- 이 Phase에서 안 할 것 (명확한 제약)

### `brand.md` (선택)
브랜드 가이드라인. 없으면 spec 에이전트가 brand 관련 린트를 SKIP합니다.
- 색상, 타이포그래피
- Voice & Tone
- 금지 표현

---

## 선택 파일

프로젝트에 특화된 규칙이 있으면 추가합니다.

**예시:**
- `lunchcoin-rules.md` — 경제 단위(코인/포인트) 규칙
- `matching-rules.md` — 매칭 알고리즘 제약
- `compliance-rules.md` — 규제·법적 제약

---

## 작성 가이드

1. `strategy.md`부터 작성. CLAUDE.md의 "전략 룰 퀵레퍼런스"와 내용이 일치해야 합니다.
2. 에이전트가 읽을 것을 전제로 작성. 간결하고 명확하게.
3. 불확실한 항목은 `[TBD]`로 표기.
