# Phase 1 PRD Lint Rules

> spec 에이전트가 PRD 저장 전 자체 검증에 사용하는 규칙 목록.
> 모든 항목 PASS 시에만 traces에 저장 가능.

---

## L1. 전략 정합성

| ID | 규칙 | PASS 조건 |
|---|---|---|
| L1-01 | 타겟 명시 | CLAUDE.md에 정의된 타겟 유저 중 하나 이상이 명시됨 |
| L1-02 | Phase 범위 | CLAUDE.md "이 Phase에서 안 할 것" 목록과 충돌 없음 |
| L1-03 | 문제 해소 연결 | 이 기능이 어떤 유저 문제를 해소하는지 명시됨 |
| L1-04 | KPI 연결 | CLAUDE.md Phase KPI 중 하나 이상에 기여함을 명시 |

---

## L2. 범위 명확화

| ID | 규칙 | PASS 조건 |
|---|---|---|
| L2-01 | Out of Scope 섹션 존재 | "이 PRD에 포함되지 않는 것" 섹션이 비어있지 않음 |
| L2-02 | 미결 항목 명시 | 결정 안 된 사항은 `[TBD]`로 표기됨 (없으면 PASS) |
| L2-03 | 성공 지표 정의 | 측정 가능한 KPI가 최소 1개 이상 명시됨 |

---

## L3. 프로젝트 특화 규칙 (해당 기능에만 적용)

> 이 섹션은 프로젝트별로 채운다.
> 예: 경제 단위(포인트, 코인 등) 규칙, 도메인 특화 제약 등.
> 해당 없으면 이 섹션 전체 삭제.

| ID | 규칙 | PASS 조건 |
|---|---|---|
| L3-01 | {규칙명} | {PASS 조건} |

---

## L4. 수락 기준 품질

| ID | 규칙 | PASS 조건 |
|---|---|---|
| L4-01 | AC 개수 | 수락 기준(AC) 최소 3개 이상 |
| L4-02 | 테스트 가능성 | 각 AC가 "Given/When/Then" 또는 동등하게 테스트 가능한 형식 |
| L4-03 | 모호한 기준 없음 | "더 나은", "빠른", "편리한" 같은 측정 불가 표현 없음 |

---

## L5. 브랜드·UX

| ID | 규칙 | PASS 조건 |
|---|---|---|
| L5-01 | CTA 단수 | Primary CTA가 1개만 제안됨 |
| L5-02 | 용어 통일 | brand.md에 정의된 금지 표현 없음 (파일 없으면 SKIP) |
| L5-03 | 브랜드 톤 충돌 없음 | brand.md에 정의된 톤과 충돌하지 않음 (파일 없으면 SKIP) |
| L5-04 | 존댓말 | 한국어 카피 예시는 해요체 사용 |

---

## 린트 결과 출력 형식

```
## Phase 1 Lint 결과

| ID | 항목 | 결과 | 비고 |
|---|---|---|---|
| L1-01 | 타겟 명시 | PASS | |
| L1-02 | Phase 범위 | PASS | |
| ...  | ...  | ...  | |

**총합:** N/N PASS
**최종:** PASS / FAIL (실패 항목: L?-??, L?-??)
```

FAIL 항목이 1개라도 있으면 수정 후 재검증. traces 저장 금지.
